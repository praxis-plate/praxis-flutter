# AGENTS.md — Praxis Flutter

This file is intentionally short. Detailed standards live in `docs/praxis_code_quality_standards.md`.
Follow the nearest `AGENTS.md` to the files you edit (directory instructions override root). 

## SDK / toolchain (FVM)

This workspace uses FVM. Always invoke Flutter/Dart via FVM:

- Use `fvm flutter ...` and `fvm dart ...`
- If a tool calls `flutter`/`dart` indirectly, run it via `fvm exec <command>`
- Do not change pinned Flutter/Dart versions unless explicitly requested

## Repository constraints

- Do not modify `packages/praxis_client` (generated by Serverpod). If changes are needed, change the source that generates it and re-generate.
- Do not commit generated build artifacts (e.g., `build/`) unless the task explicitly requires it.

## Default commands

- Install: `fvm flutter pub get`
- Format: `fvm dart format .`
- Lint: `fvm dart analyze`
- Tests: `fvm flutter test`
- Run: `fvm flutter run`

If a task requires additional commands, prefer documenting them in a closer `AGENTS.md` (e.g., `test/AGENTS.md`) rather than expanding this file.

## Definition of done

- Formatting and analysis pass (or explain why they cannot).
- Tests relevant to the change pass.
- Clean Architecture boundaries are preserved (see `lib/AGENTS.md`).
- Changes are minimal and scoped; avoid drive-by refactors.

## Review guidelines

- Reject boundary violations (Domain depending on Data/Flutter, Repositories doing DB work, etc.).
- Prefer small, coherent diffs with clear intent.
- Require tests for new/changed business logic and bug fixes.
- Avoid adding new dependencies without a clear need.
